프로젝트 내의 권한은 조직 내 권한과 별도로 구성되어있습니다.

프로젝트 권한은 프로젝트 내에서만 적용되고, 조직 소유자나 조직 권한 `Admin` 이상의 멤버는 프로젝트 권한이나 프로젝트 멤버 여부에 관계없이 프로젝트 내에서 모든 권한을 가집니다.

프로젝트의 권한은 다음과 같이 구성되어 있습니다.

- `Admin`: 프로젝트의 모든 권한을 가집니다.
- `Write`: 프로젝트의 모든 권한을 가지지만, 프로젝트의 멤버, 설정을 변경할 수 없습니다.
- `Read`: 테스트 결과를 볼 수 있습니다.

## 권한 비교

### 프로젝트 관리

| 권한                     | Admin | Write | Read |
| ------------------------ | ----- | ----- | ---- |
| 프로젝트 ID 조회         | ✅    | ✅    | ✅   |
| 프로젝트 Git 연동        | ✅    | ❌    | ❌   |
| 프로젝트 이름, 설명 변경 | ✅    | ❌    | ❌   |

:::info 프로젝트 삭제

프로젝트 삭제는 프로젝트 권한과 관계 없이 조직 소유자 또는 `Admin` 권한 이상의 멤버만 가능합니다.

<a href="../organization/project#프로젝트-삭제하기">프로젝트 삭제하기</a> 항목을
참고하세요.

:::
