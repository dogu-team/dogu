# 디바이스 관리

:::info PREREQUISTIES

이 문서는 호스트가 1대 이상 이미 연결을 마친 상태에서 진행됩니다.

:::

## 디바이스 추가하기

<OrganizationAdminPermission prefix="디바이스 추가는" />

디바이스를 추가하려면 호스트와 디바이스를 물리적으로 연결 후 콘솔에서 추가할 수 있습니다.

### 디바이스와 호스트 연결하기

:::info

디바이스와 호스트를 연결하는 방법은 [디바이스 설정](/device-farm/device/settings)를 참고해주세요.

:::

### 콘솔에서 디바이스 사용하기

:::info PREREQUISTIES

이 파트는 디바이스가 호스트와 연결된 상태에서 진행됩니다.

:::

1. Dogu 콘솔에서 조직 페이지 내 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭합니다.
2. "대기중인 디바이스" 탭 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/add-device-menu.png"
     style={{ width: 600, display: 'block' }}
   />
3. 목록에서 추가할 디바이스의 메뉴 버튼(`···`)을 클릭합니다.
4. "사용하기"를 클릭합니다.
   <img
     src="/ko/img/console/organization/device-start-using.png"
     style={{ width: 600, display: 'block' }}
   />
5. 해당 디바이스를 사용할 프로젝트를 선택합니다.(다수 선택 가능)

   <img
     src="/ko/img/console/organization/device-alloc-project.png"
     style={{ width: 600, display: 'block' }}
   />

   <br />

   :::tip 퍼블릭 디바이스

   만약 프로젝트 상관없이 사용하려면, 퍼블릭 디바이스로 선택해
   추가하세요. 단, 모든 멤버가 사용할 수 있으므로 주의하세요.

   :::

6. 프로젝트 선택이 끝나면, 모달을 닫고 "사용중인 디바이스" 탭 메뉴로 이동합니다.
   <img
     src="/ko/img/console/organization/device-alloc-project-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
7. 추가한 디바이스가 사용중인 디바이스 목록에 추가된 것을 확인할 수 있습니다.
   <img
     src="/ko/img/console/organization/device-add-complete.png"
     style={{ width: 600, display: 'block' }}
   />

:::tip

사용 대기중인 디바이스가 생기면 "대기중인 디바이스" 탭 메뉴에 빨간 점이 표시됩니다.

:::

:::info 트러블 슈팅

디바이스 연결에 문제가 생겼나요? [디바이스 트러블 슈팅](/device-farm/device/trouble-shooting) 문서를 참고해주세요.

:::

### 디바이스 상태

디바이스 상태는 크게 2가지가 있습니다.

**디바이스 연결상태**

| 항목      | 설명                                                                                            |
| --------- | ----------------------------------------------------------------------------------------------- |
| `Online`  | 호스트와 연결되었고, 콘솔에도 연결된 상태                                                       |
| `Offline` | 호스트와 연결되었으나, 콘솔에 연결되지 않은 상태 / 호스트와 연결되지 않은 상태                  |
| `Unknown` | 호스트와 연결되었으나, 콘솔에 연결되지 않은 상태 / 호스트와 연결되지 않은 상태 / 기타 장애 상태 |

**디바이스 사용상태**

| 항목      | 설명                                                                       |
| --------- | -------------------------------------------------------------------------- |
| `Idle`    | 테스트나 루틴에서 사용하지 않는 유휴 상태                                  |
| `Running` | 테스트나 루틴에서 사용 중인 상태. 디바이스에서 테스트가 수행되고 있습니다. |
| `Unknown` | 디바이스의 사용 상태를 알 수 없는 상태                                     |

## 호스트 디바이스 러너

호스트 디바이스 러너는 호스트 디바이스에서 실행되는 러너입니다. 호스트 디바이스는 1~16개의 러너를 가질 수 있습니다.  
러너는 하나의 작업을 수행하며, 웹 테스트에서 하나의 브라우저를 의미합니다.

<img
  src="/ko/img/console/organization/host-device-runners.png"
  style={{ width: 600, display: 'block' }}
/>

:::info

러너는 호스트 디바이스에서만 존재합니다.

:::

### 러너 수 변경하기

1. Dogu 콘솔에서 조직 페이지 내 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭합니다.
2. "사용중인 디바이스" 또는 "대기중인 디바이스" 탭 메뉴를 클릭합니다.
3. 러너 수를 변경할 호스트 디바이스의 메뉴 버튼(`···`)을 클릭합니다.
   <img
     src="/ko/img/console/organization/host-device-runner-setting-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. "러너 설정" 메뉴를 클릭합니다.
5. 러너 수를 변경하고 저장 버튼을 클릭합니다.
   <img
     src="/ko/img/console/organization/host-device-runner-setting-modal.png"
     style={{ width: 600, display: 'block' }}
   />

## 디바이스 태그 변경하기

<OrganizationAdminPermission prefix="디바이스 태그 변경은" />

### 디바이스에 태그 붙이기

:::tip 플랫폼 태그

플랫폼 태그는 디바이스를 사용할 때 자동으로 추가됩니다.

:::

1. Dogu 콘솔에서 조직 페이지 내 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭합니다.
2. "사용중인 디바이스" 탭에서 태그를 변경하고자 하는 디바이스를 찾고, 메뉴 버튼(`···`)을 클릭합니다.
3. "태그 변경" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/edit-device-tag-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. 변경할 태그를 선택합니다.
   - 추가할 태그가 없는 경우, 태그를 만들 수 있습니다.
     <img
       src="/ko/img/console/organization/attach-device-tag-with-creation.png"
       style={{ width: 600, display: 'block' }}
     />
   - 기존에 태그가 이미 있는 경우, 검색 후 태그를 클릭합니다.
     <img
       src="/ko/img/console/organization/attach-device-tag-find.png"
       style={{ width: 600, display: 'block' }}
     />
5. 태그가 추가된 것을 확인하고, 모달을 닫습니다.
   <img
     src="/ko/img/console/organization/attach-device-tag-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
6. 디바이스 목록에서 태그가 추가된 것을 확인합니다.
   <img
     src="/ko/img/console/organization/attach-device-tag-count-confirm.png"
     style={{ width: 600, display: 'block' }}
   />

### 디바이스에 태그 떼기

:::tip 플랫폼 태그

플랫폼 태그는 디바이스에서 땔 수 없습니다.

:::

1. Dogu 콘솔에서 조직 페이지 내 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭합니다.
2. "사용중인 디바이스" 탭에서 태그를 변경하고자 하는 디바이스를 찾고, 메뉴 버튼(`···`)을 클릭합니다.
3. "태그 변경" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/edit-device-tag-menu-remove.png"
     style={{ width: 600, display: 'block' }}
   />
4. 떼고자 하는 태그에서 X 버튼을 클릭합니다.
   <img
     src="/ko/img/console/organization/detach-device-tag-modal.png"
     style={{ width: 600, display: 'block' }}
   />
5. 디바이스 목록에서 태그가 떼어진 것을 확인합니다.
   <img
     src="/ko/img/console/organization/detach-device-tag-confirm.png"
     style={{ width: 600, display: 'block' }}
   />

:::danger

안전한 사용중지를 위해 디바이스가 수행하는 테스트가 없는지 확인 후에 사용 중지를 진행해주세요.

디바이스를 사용 중지하게 되면 디바이스와 프로젝트 간의 연결이 끊기게 됩니다.<br />
중지된 디바이스는 '디바이스 추가' 메뉴에서 다시 활성화할 수 있습니다.<br />
다시 활성화된 디바이스는 이전에 등록된 태그가 유지됩니다.

:::

1. Dogu 콘솔에서 조직 페이지 내 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭합니다.
2. "사용중인 디바이스" 탭에서 사용 중지하고자 하는 디바이스를 찾고, 메뉴 버튼(`···`)을 클릭합니다.
3. "사용 중지" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/stop-using-device-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. 확인 버튼을 클릭하고, "디바이스 추가" 탭으로 이동해 디바이스가 사용 중지된 것을 확인합니다.
