조직에 속한 호스트를 관리합니다.

## 호스트 추가하기

<OrganizationAdminPermission prefix="호스트 추가는" />

:::info PREREQUISTIES

호스트 장비에 Dogu Agent가 설치되어 있어야 합니다.

<a href="https://dogutech.io/downloads/dogu-agent">Dogu Agent 다운로드</a> 페이지에서
Dogu Agent를 다운로드할 수 있습니다.

호스트 설정 및 사양에 대한 내용은 [Agent](/device-farm/host) 문서를 참고하세요.

:::

1. Dogu Agent를 설치한 호스트 장비를 네트워크에 연결합니다.
2. 호스트 장비에서 Dogu Agent를 실행합니다.
3. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
4. "새 호스트 생성" 버튼을 클릭합니다.
5. 호스트 이름을 입력하고 "생성" 버튼을 클릭합니다.
   <img
     src="/ko/img/console/organization/add-host-modal.png"
     style={{ width: 600, display: 'block' }}
   />
6. 호스트 토큰을 Dogu Agent의 토큰 입력창에 입력하고, "Connect" 버튼을 클릭합니다.
   <img
     src="/ko/img/get-started/dost-token-input.png"
     style={{ width: 600, display: 'block' }}
   />
7. Dogu 콘솔에서 호스트가 추가되었는지 확인합니다.
   ![host connected](/img/console/organization/host-connected.png)

## 호스트 정보

<OrganizationMemberPermission prefix="호스트 정보 조회는" />

### 호스트 목록

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 목록의 각 열은 다음과 같은 정보를 담고 있습니다
   ![host list](/img/console/organization/host-list.png)

| 항목 | 이름        | 연결상태           | OS                | 디바이스 수                 |
| ---- | ----------- | ------------------ | ----------------- | --------------------------- |
| 설명 | 호스트 이름 | 호스트의 연결 상태 | 호스트의 운영체제 | 호스트에 연결된 디바이스 수 |

:::info OS 아이콘

호스트의 운영체제에 따라 아이콘이 표기됩니다.

- Windows
- macOS

호스트를 연결한 적이 없거나, 호스트의 운영체제가 Windows 또는 macOS가 아닌 경우에는 별도의 아이콘이 표기됩니다.

:::

### 호스트 연결 상태

호스트의 연결 상태는 다음과 같습니다.

- `Connected` : 호스트가 콘솔에 연결되어 있습니다.
- `Disconnected` : 호스트가 콘솔에 연결되어 있지 않습니다.
- `Unknown` : 호스트가 콘솔에 연결되어 있지 않습니다. 장애가 발생해 연결 상태를 콘솔에서 확인이 불가능한 상태입니다.

### 호스트 상세 정보

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 상세정보를 볼 호스트 이름을 클릭합니다.
   <img
     src="/ko/img/console/organization/host-detail-modal.png"
     style={{ width: 500, display: 'block' }}
   />

| 항목            | 설명                                                                            |
| --------------- | ------------------------------------------------------------------------------- |
| 이름            | 호스트 이름                                                                     |
| 연결 날짜       | 호스트가 추가된 날짜                                                            |
| 연결된 디바이스 | 호스트에 연결된 디바이스 리스트 및 디바이스 수                                  |
| 연결한 멤버     | 호스트를 추가한 멤버                                                            |
| 호스트 토큰     | 기존 호스트 토큰. 조직 소유자 및 `Admin` 권한 이상의 멤버만 확인할 수 있습니다. |

## 호스트 디바이스 사용하기

<OrganizationAdminPermission prefix="호스트 디바이스 사용은" />

호스트 또한 디바이스로 사용할 수 있습니다. 디바이스로 사용하면 호스트 디바이스로 루틴, 스크립트 테스트를 수행할 수 있습니다.

:::info

호스트에 연결된 디바이스는 영향 없이 사용할 수 있습니다.

:::

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 디바이스로 사용할 호스트를 찾고 메뉴 버튼(`···`)을 클릭하세요.
3. "디바이스로 사용하기" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/host-use-as-device.png"
     style={{ width: 500, display: 'block' }}
   />
4. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스" 메뉴를 클릭합니다.
5. "대기중인 디바이스"에서 `Host` 태그가 붙은 디바이스가 목록에 나타났는지 확인합니다.
   <img
     src="/ko/img/console/organization/chekc-host-use-as-device.png"
     style={{ width: 500, display: 'block' }}
   />

## 호스트 디바이스 사용 중지하기

<OrganizationAdminPermission prefix="호스트 디바이스 사용 중지는" />

:::danger

호스트를 디바이스로 사용 중지하면 진행하고 있는 루틴, 스크립트 테스트가 실패할 수 있습니다.

호스트 디바이스를 사용하고 있는 곳이 없는지 확인하고 사용 중지하세요.

:::

:::info

호스트에 연결된 디바이스는 영향 없이 사용할 수 있습니다.

:::

호스트 또한 디바이스로 사용할 수 있습니다. 디바이스로 사용하면 호스트 디바이스로 루틴, 스크립트 테스트를 수행할 수 있습니다.

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 디바이스로 사용을 중지할 호스트를 찾고 메뉴 버튼(`···`)을 클릭하세요.
3. "디바이스로 사용 중지하기" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/host-stop-use-as-device.png"
     style={{ width: 500, display: 'block' }}
   />

## 호스트 토큰 재발급

<OrganizationAdminPermission prefix="호스트 토큰 재발급은" />

:::danger 토큰 재발급

호스트의 토큰을 재발급하면, 기존 호스트 토큰은 더 이상 사용할 수 없습니다. 따라서 Dogu Agent에 재발급된 토큰을 입력하기 전까지는 호스트의 연결상태가 `Disconnected`가 됩니다.

해당 호스트에 연결된 디바이스 또한 연결상태가 `Offline`이 되므로, 진행되고 있는 테스트나 디바이스 원격제어가 중단될 수 있습니다.

:::

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 재발급할 호스트를 찾고 메뉴 버튼(`···`)을 클릭하세요.
3. "토큰 재발급" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/revoke-host-token.png"
     style={{ width: 600, display: 'block' }}
   />
4. 확인버튼을 클릭 후 재발급된 토큰을 복사합니다.
   <img
     src="/ko/img/console/organization/revoke-host-token-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
5. Dogu Agent를 실행한 호스트 장비에서 'Troubleshoot' 탭을 클릭합니다.
   <img
     src="/ko/img/console/organization/dost-troubleshoot-tab.png"
     style={{ width: 600, display: 'block' }}
   />
6. 'Reconnect' 버튼을 클릭합니다.
   <img
     src="/ko/img/console/organization/dost-reconnect.png"
     style={{ width: 600, display: 'block' }}
   />
7. 재발급된 토큰을 입력하고 'Connect' 버튼을 클릭합니다.
   <img
     src="/ko/img/get-started/dost-token-input.png"
     style={{ width: 600, display: 'block' }}
   />

## 호스트 삭제하기

<OrganizationAdminPermission prefix="호스트 삭제는" />

:::danger 연결된 디바이스

호스트를 삭제하면 호스트에 연결된 디바이스가 모두 삭제됩니다. 테스트가 실행되고 있는 디바이스가 있는지 확인하고 삭제해 주세요.

디바이스 삭제는 `디바이스 사용 중지하기`와는 다르게 디바이스와 관련된 정보를 완전히 삭제합니다. <br />
해당 디바이스의 정보를 유지하고 싶다면 조직 내의 다른 호스트로 연결 후 해당 호스트를 삭제해 주세요.

:::

1. Dogu 콘솔에서 조직 페이지의 왼쪽 사이드바에서 "디바이스 팜" 메뉴를 클릭하고 "호스트" 탭을 클릭합니다.
2. 호스트 목록에서 삭제할 호스트를 찾고 메뉴 버튼(`···`)을 클릭하세요.
3. "호스트 삭제" 메뉴를 클릭합니다.
   <img
     src="/ko/img/console/organization/host-delete-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. 확인 버튼을 누르면 호스트가 삭제됩니다.
