조직의 권한은 다음과 같이 구성됩니다.

- 조직 소유자
  - 조직을 생성한 사용자는 조직의 소유자가 됩니다.
- `Admin`
- `Member`

## 권한 비교

### 조직 관리

| 항목                        | 조직 소유자 | Admin | Member |
| --------------------------- | ----------- | ----- | ------ |
| 조직 ID 조회                | ✅          | ✅    | ✅     |
| Project Access Token 조회   | ✅          | ✅    | ❌     |
| Project Access Token 재발급 | ✅          | ✅    | ❌     |
| 조직 삭제                   | ✅          | ❌    | ❌     |
| 조직 소유자 변경            | ✅          | ❌    | ❌     |
| 결제 관리                   | ✅          | ❌    | ❌     |
| 조직 프로필 변경            | ✅          | ❌    | ❌     |

:::info

조직 소유자와 <code>Admin</code> 권한 멤버는 모든 프로젝트를 조회할 수 있습니다.

<code>Member</code> 권한 멤버는 자신이 속한 프로젝트만 조회할 수 있고, 프로젝트 내
활동 권한은 <a href={'../project/permission'}>프로젝트 권한 시스템</a> 페이지를 참고하세요.

:::

:::info

<code>Member</code> 권한의 멤버의 경우, 프로젝트에 할당된 디바이스는 해당 프로젝트에
속한 멤버만 스트리밍 및 원격제어가 가능합니다.

:::
