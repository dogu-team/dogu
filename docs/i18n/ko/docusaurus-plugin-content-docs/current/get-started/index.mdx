Dogu는 웹, 모바일, 게임 애플리케이션을 다양한 운영 체제에서 테스트를 자동화할 수 있도록 돕는 통합 테스트 자동화 플랫폼입니다.  
통합 테스트 자동화 플랫폼을 이용하여 기존의 인프라 구축 과정을 거치지 않고 효율적인 테스트 자동화 환경을 구축할 수 있습니다.

## 기능

체계적인 테스트 자동화 환경을 구축하기 위해 필요한 기능을 제공합니다.

### 관리

관리 기능을 이용하여 사용자 및 그룹을 생성하고, 권한을 부여할 수 있으며 권한 계층은 아래와 같이 형성돼 있으며 권한 계층에 따라 접근 범위가 다르며 필요한 경우 권한을 별도로 부여할 수 있습니다.

- 조직
  - 프로젝트
    - 사용자
    - 사용자 그룹

### 디바이스 팜

디바이스 팜은 테스트를 실행하고 상호작용할 수 있는 디바이스가 모인 플랫폼입니다. 디바이스 팜에 연결할 수 있는 운영체제는 Windows, MacOS, Android, iOS가 있으며, 디바이스 팜에 연결된 디바이스와 상호작용할 수 있습니다.

### 테스트 자동화

디바이스 팜에 연결된 디바이스를 대상으로 브라우저, 모바일, 게임 테스트를 자동화할 수 있는 테스트 프레임워크와 함께 사용할 수 있습니다.  
지원하는 테스트 프레임워크에 대해 자세히 알아보려면 아래 항목을 참고하시기 바랍니다.

#### 테스트 프레임워크 지원 목록

- [브라우저](/ko/test-automation/browser)
- [모바일](/ko/test-automation/mobile)
- [게임](/ko/test-automation/game)

테스트 프레임워크를 이용해 디바이스 팜에 연결된 디바이스를 대상으로 테스트를 실행할 수 있는 방법은 아래와 같이 2가지가 있습니다.

- [Relay](/ko/script-and-routine/routine/about)
- [Routine](/ko/script-and-routine/routine/about)

#### Relay

Relay 방식은 로컬에서 작업중인 테스트 스크립트를 Relay Hub를 통해 디바이스 팜에 연결된 디바이스를 대상으로 테스트를 실행하는 것을 의미합니다.  
Relay를 통해 대기중인 디바이스를 자동으로 할당받아 테스트를 실행할 수 있으며, 테스트가 종료되면 디바이스를 자동으로 반납합니다.

#### Rotuine

Routine은 Github Action, Jenkins Pipeline과 유사한 테스트를 위한 Pipeline 입니다.
테스트 스크립트가 Git에 업로드되면 Routine을 통해 해당 테스트 스크립트를 선택하고 테스트를 실행할 디바이스를 지정하여 테스트를 실행할 수 있습니다.

<img src="/ko/img/get-started/routine.png" />

### 테스트 리포트

테스트 리포트는 Routine을 통해 실행된 테스트 결과를 확인할 수 있습니다. 테스트 결과에는 아래 항목이 포함됩니다.

- 테스트 실행 영상
- 테스트 스크립트 시각화
- 디바이스 로그 및 테스트 스크립트 로그
- 디바이스 프로파일링 (Memory, CPU, FPS)

리포트를 작성하기 위해서는 Test Unit Framework에 맞는 플러그인을 설치해야 합니다. 지원하는 플러그인 목록은 아래와 같습니다.

- [Jest](/ko/test-report/javascript/jest)
- [PyTest](/ko/test-report/python/pytest)
- TestNG - 개발중

플러그인을 설치하고 테스트를 실행하면 아래와 같이 테스트 리포트가 생성됩니다.

### 루틴

루틴은 테스트를 체계적으로 실행할 수 있는 프로세스를 의미합니다. 조건에 따른 디바이스를 선택하여 테스트를 병렬로 실행할 수 있으며 프로세스 후처리 작업을 지원합니다. 프로세스가 종료되면 테스트 리포트가 자동으로 생성됩니다.
