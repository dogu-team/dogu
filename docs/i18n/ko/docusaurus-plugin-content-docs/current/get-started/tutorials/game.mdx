:::info PREREQUISITES

이 문서는 디바이스 팜이 구축된 이후 게임 테스트를 시작하는 방법에 대해 설명합니다.

아직 디바이스 팜을 구축하지 않았다면, [디바이스 팜 시작하기](/get-started/tutorials/device-farm)를 참고해 디바이스 팜을 구축해 주세요.

:::

## 샘플 게임 다운로드

Dogu에서 제공하는 샘플 게임을 다운로드 받아 게임 테스트를 시작해 보세요.
샘플 게임은 [Dogu RPG Sample](https://github.com/dogu-team/gamium-unity-samples/releases?q=DoguRpgSample&expanded=true)에서 다운로드 받으세요.

1. 위 링크를 클릭합니다.
2. `android.zip` 또는 `ios.zip` 파일을 다운로드 받습니다.
   <img
     src="/ko/img/get-started/download-sample-game.png"
     style={{ width: 600, display: 'block' }}
   />
3. 압축을 풀어 `dogurpgsample` 앱 파일을 확인합니다.

## 프로젝트에 앱 업로드하기

1. "앱" 메뉴 탭을 클릭합니다.
2. "앱 업로드" 버튼을 클릭합니다.
   <img
     src="/ko/img/console/project/upload-app.png"
     style={{ width: 600, display: 'block' }}
   />
3. `dogurpgsample` 앱 파일을 업로드합니다.
   <img
     src="/ko/img/console/project/app-upload-complete.png"
     style={{ width: 600, display: 'block' }}
   />

## 테스트 스크립트 작성 및 Git 연동하기

:::info

Dogu는 GitHub, GitLab 연동을 지원합니다.

:::

1. [Gamium sample](https://github.com/dogu-team/dogu-examples/tree/main/typescript/gamium) GitHub 레포지토리를 참고해 테스트 스크립트를 작성해 보세요.
2. [Git 연동하기](/management/project/git-integration) 문서를 참고해 프로젝트와 Git 레포지토리를 연동해 보세요.

## 루틴 만들기

게임 테스트를 루틴으로 만들어 실행할 수 있습니다. 루틴을 통해 체계적이고 반복적인 게임 테스트를 실행해 보세요.

1. 프로젝트 페이지에서 "루틴" 탭을 클릭 후, 왼쪽 사이드바의 "+" 버튼을 클릭합니다.
   <img
     src="/ko/img/console/project/create-routine.png"
     style={{ width: 600, display: 'block' }}
   />
2. 작업 내 디바이스 태그 항목에서 "+" 버튼을 클릭하고, 안드로이드 디바이스에 추가한 태그를 선택합니다.
   <img
     src="/ko/img/get-started/routine-tag.png"
     style={{ width: 600, display: 'block' }}
   />
3. 디바이스 화면 녹화 옵션을 활성화 합니다.
   <img
     src="/ko/img/get-started/routine-record.png"
     style={{ width: 600, display: 'block' }}
   />
4. `prepare` 스텝에서 이전에 업로드한 앱을 선택합니다.
   <img
     src="/ko/img/get-started/routine-prepare.png"
     style={{ width: 600, display: 'block' }}
   />
5. `run test` 스텝에서 `dogurpgsample.test.ts` 파일을 선택합니다.
   <img
     src="/ko/img/get-started/routine-script.png"
     style={{ width: 600, display: 'block' }}
   />
6. "저장" 버튼을 클릭해 루틴을 만듭니다.

:::info 디바이스 화면 녹화

디바이스에서 해당 작업을 실행할 때 화면을 녹화할지 여부를 지정할 수 있습니다. 녹화된 영상은 해당 디바이스 작업이 종료되면 결과 페이지에서 확인할 수 있습니다.

:::

## 루틴 실행하기

1. 왼쪽 사이드바에서, 생성된 루틴을 클릭합니다.
2. "루틴 실행" 버튼을 클릭하면, 루틴이 실행되면서 파이프라인이 생성됩니다.

   <img
     src="/ko/img/get-started/routine-run.png"
     style={{ width: 600, display: 'block' }}
   />
   <br />

   :::caution 디바이스 연결상태

   루틴이 실행되지 않았다면, 디바이스의 연결 상태가 `Online` 인지 확인하세요. 만약 `Offline` 상태라면, [디바이스 팜 시작하기](/get-started/tutorials/device-farm) 단계에서 디바이스를 연결하는 과정을 다시 확인하거나, [디바이스 트러블슈팅](/device-farm/device/trouble-shooting) 문서를 참고하세요.

   :::

3. 생성된 파이프라인을 클릭합니다.
   <img
     src="/ko/img/get-started/pipeline-created.png"
     style={{ width: 600, display: 'block' }}
   />

## 파이프라인 결과 확인하기

파이프라인이 진행중이거나, 완료되면 디바이스 작업 결과를 확인할 수 있습니다.

1. 왼쪽 사이드바에서 작업을 클릭하고, 디바이스 작업을 선택합니다.
   <img
     src="/ko/img/routine/pipeline-live-sidebar.png"
     style={{ width: 600, display: 'block' }}
   />
2. 테스트 결과를 확인합니다. 녹화 영상, 로그, 테스트 스크립트로 생성된 리포트, 디바이스 프로파일 정보 등을 확인할 수 있습니다.

:::tip

테스트 결과에 대한 자세한 정보는 [테스트 리포트](/test-report) 문서를 참고하세요.

:::

## 마치며

게임 테스트가 잘 수행되었나요? 샘플 게임 외에 다른 게임을 테스트해 보세요.

1. Dogu 콘솔에서 새 프로젝트를 생성합니다.
2. [Gamium Engine SDK 가이드](https://gamium.dogutech.io/docs/engine/unity/project-configuration)를 참고해 테스트할 게임에 Gamium Engine SDK를 추가하고 빌드 합니다.
3. [루틴 가이드](/routine/routines)를 참고하고, 루틴을 만들어 실행하고 결과를 확인해보세요.
