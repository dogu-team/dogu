## Create Pipeline

```
POST /v1/projects/:projectId/routines/:routineId/pipelines
```

### Query Parameter

| Attribute   | type   | required | description                                                                                                                                 |
| ----------- | ------ | -------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `projectId` | string | true     | ID of the project. You can easily find it on the project settings page. See [the project setup documentation](/management/project/settings) |
| `routineId` | string | true     | ID of the Routine. Refer to [Check Routine ID](/management/project/routine#check-routine-id)                                                |

### Header

| Attribute       | type   | required | description                                                                                      |
| --------------- | ------ | -------- | ------------------------------------------------------------------------------------------------ |
| `Authorization` | string | true     | Authorization: Bearer ${ACCESS_TOKEN}.<br /> For ACCESS_TOKEN, refer to the [ACCESS TOKEN](/api) |

### Example

```
curl -v -X POST "https://api.dogutech.io/v1/projects/${projectId}/routines/${routineId}/pipelines" \
-H "Authorization: Bearer ${ACCESS_TOKEN}" \
-H "Content-Type: application/json"
```

## Get Pipeline

```
GET /v1/projects/:projectId/routines/:routineId/:routineId/pipelines/:routinePipelineId
```

### Query Parameter

| Attribute           | type   | required | description                                                                                                                                                                                                   |
| ------------------- | ------ | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `projectId`         | string | true     | ID of the project. You can easily find it on the project settings page. See [the project setup documentation](/management/project/settings)                                                                   |
| `routineId`         | string | true     | ID of the Routine. Refer to [Check Routine ID](/management/project/routine#check-routine-id)                                                                                                                  |
| `routinePipelineId` | string | true     | ID of the Pipeline. Refer to [CreatePipeline Response](/api/routine/pipeline#create-pipeline-response) <br/> Or Check in a similar manner to [Check Routine ID](/management/project/routine#check-routine-id) |

### Header

| Attribute       | type   | required | description                                                                                      |
| --------------- | ------ | -------- | ------------------------------------------------------------------------------------------------ |
| `Authorization` | string | true     | Authorization: Bearer ${ACCESS_TOKEN}.<br /> For ACCESS_TOKEN, refer to the [ACCESS TOKEN](/api) |

### Example

```
curl -v -X GET "https://api.dogutech.io/v1/projects/${projectId}/routines/${routineId}/pipelines/${routinePipelineId}" \
-H "Authorization: Bearer ${ACCESS_TOKEN}" \
-H "Content-Type: application/json"
```
