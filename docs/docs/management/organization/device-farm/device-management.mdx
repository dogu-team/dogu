## Add a device

<OrganizationAdminPermission prefix="Adding a device" />

To add a device, you need to physically connect it to the host and then you can add it in the console.

### Connect a device to a host

:::info

Please refer to the [Device Settings](/device-farm/device/settings) for instructions on how to connect a device to a host.

:::

### Start using device from console

:::info PREREQUISTIES

This part proceeds with the device connected to the host.

:::

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. Click on the "Standby devices" tab.
   <img
     src="/img/console/organization/add-device-menu.png"
     style={{ width: 600, display: 'block' }}
   />
3. Click on the menu button(`···`) of the device you want to add from the list.
4. Click on "Start using".
   <img
     src="/img/console/organization/device-start-using.png"
     style={{ width: 600, display: 'block' }}
   />
5. Select the project(s) in which you want to use the device. (Multiple selections are possible)

   <img
     src="/img/console/organization/device-alloc-project.png"
     style={{ width: 600, display: 'block' }}
   />

   <br />

   :::tip Public Devices

   If you want to use the device without assigning it to a specific project, select it as a Public Device. However, please note that all members will be able to use it.

   :::

6. Once you have selected the project(s), close the modal and navigate to the "In-use devices" tab.
   <img
     src="/img/console/organization/device-alloc-project-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
7. You will see that the added device is now listed under the "In-use devices" section.
   <img
     src="/img/console/organization/device-add-complete.png"
     style={{ width: 600, display: 'block' }}
   />

:::tip

When there are pending devices, a red dot will appear on the "Standby devices" tab.

:::

:::info Trouble Shooting

Having trouble with device connections? Please refer to the [Device Troubleshooting](/device-farm/device/trouble-shooting) document for assistance.

:::

## Device information

<OrganizationMemberPermission prefix="Viewing device list" />

### Device status

There're two types of device status: `Connection status` and `Usage status`.

**Connection status**

| Status    | Description                                                                                       |
| --------- | ------------------------------------------------------------------------------------------------- |
| `Online`  | Connected to the host and also connected in the console                                           |
| `Offline` | Connected to the host but not connected in the console / Not connected to the host                |
| `Unknown` | Connected to the host but not connected in the console / Not connected to the host / Other issues |

**Usage status**

| Status    | Description                                                                                  |
| --------- | -------------------------------------------------------------------------------------------- |
| `Idle`    | Device is in an idle state and not being used in tests or routines                           |
| `Running` | Device is currently being used in tests or routines. Tests are being executed on the device. |
| `Unknown` | The usage status of the device cannot be determined.                                         |

## Host device runner

The host device runner is a runner that runs on the host device. A host device can have 1 to 16 runners.  
A runner performs a single task and represents one browser in web testing.

<img
  src="/img/console/organization/host-device-runners.png"
  style={{ width: 600, display: 'block' }}
/>

:::info

Runners exist only on the host device.

:::

### Changing the Number of Runners

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. Click on the "In-use devices" or "Standby devices" tab menu.
3. Click on the menu button (`···`) of the host device for which you want to change the number of runners.
   <img
     src="/img/console/organization/host-device-runner-setting-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. Click on the "Runner settings" menu.
5. Change the number of runners and click the save button.
   <img
     src="/img/console/organization/host-device-runner-setting-modal.png"
     style={{ width: 600, display: 'block' }}
   />

## Update device's tags

<OrganizationAdminPermission prefix="Updating device's tags" />

### Attach tags to a device

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. In the "In-use deviecs" tab, find the device to which you want to add tags and click on the menu button(`···`).
3. Click on the "Edit tags" menu.
   <img
     src="/img/console/organization/edit-device-tag-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. Select the tags you want to add to the device.
   - If there are no existing tags to add, you can create a new tag.
     <img
       src="/img/console/organization/attach-device-tag-with-creation.png"
       style={{ width: 600, display: 'block' }}
     />
   - If there are existing tags, search for the tag and click on it.
     <img
       src="/img/console/organization/attach-device-tag-find.png"
       style={{ width: 600, display: 'block' }}
     />
5. Confirm that the tags have been added and close the modal.
   <img
     src="/img/console/organization/attach-device-tag-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
6. Check that the tags have been added to the device in the device list.
   <img
     src="/img/console/organization/attach-device-tag-count-confirm.png"
     style={{ width: 600, display: 'block' }}
   />

### Detach tags from a device

:::tip Platform Tags

Platform tags cannot be removed from a device.

:::

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. In the "In-use devices" tab, find the device from which you want to remove tags and click on the menu button(`···`).
3. Click on the "Edit tags" menu.
   <img
     src="/img/console/organization/edit-device-tag-menu-remove.png"
     style={{ width: 600, display: 'block' }}
   />
4. Click on the "X" button next to the tag you want to remove.
   <img
     src="/img/console/organization/detach-device-tag-modal.png"
     style={{ width: 600, display: 'block' }}
   />
5. Check that the tag has been removed from the device in the device list.
   <img
     src="/img/console/organization/detach-device-tag-confirm.png"
     style={{ width: 600, display: 'block' }}
   />

## Adding projects to a device

<OrganizationAdminPermission prefix="Adding projects to a device" />

:::caution Note

If a device is not used in any projects, it will be automatically deactivated.

You must either use the device in at least one project or set it as a public device.

:::

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. In the "In-use devices" tab, find the device to which you want to add projects and click on the menu button(`···`).
3. Click on the "Edit projects" menu.
   <img
     src="/img/console/organization/edit-device-project-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. Search for the project you want to add and click on the project.
   <img
     src="/img/console/organization/attach-device-project-find.png"
     style={{ width: 600, display: 'block' }}
   />
5. Check that the project has been added and close the modal.
   <img
     src="/img/console/organization/attach-device-project-confirm.png"
     style={{ width: 600, display: 'block' }}
   />
6. Check that the project has been added to the device in the device list.
   <img
     src="/img/console/organization/attach-device-project-count-confirm.png"
     style={{ width: 600, display: 'block' }}
   />

   <img
     src="/img/console/organization/device-setting-menu.png"

   />

## Stop using a device

<OrganizationAdminPermission prefix="Stop using a device" />

:::danger Warning

Please ensure that there are no ongoing tests being performed by the device before deactivating it.

When you deactivate a device, it will be disconnected from any projects it was associated with.
A deactivated device can be reactivated using the "Add Device" menu.
Reactivated devices will retain their previously registered tags.

:::

1. In the Dogu console, click on the "Device Farm" menu in the left sidebar of the organization page.
2. In the "In-use devices" tab, find the device you want to stop using and click on the menu button(`···`).
3. Click on the "Stop using" menu.
   <img
     src="/img/console/organization/stop-using-device-menu.png"
     style={{ width: 600, display: 'block' }}
   />
4. Click on the confirm button and click on the "Standby devices" tab to check that the device has been deactivated.
